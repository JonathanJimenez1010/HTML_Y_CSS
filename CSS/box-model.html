Guía completa del Box Model y valores importantes de display (HTML/CSS)

Explicación detallada, ejemplos visuales y notas prácticas — todo en un solo archivo HTML para experimentar.

Introducción rápida

En CSS cada elemento se representa como una caja (box). El Box Model define cómo se calculan el tamaño y el espacio alrededor de esa caja. Entenderlo es crucial para controlar diseños, evitar problemas de colapso y crear interfaces consistentes.

Componentes del Box Model

  1. content — el área donde va el texto o contenido.
  2. padding — espacio entre el contenido y el borde (interior).
  3. border — borde alrededor del padding (puede tener grosor, estilo y color).
  4. margin — espacio exterior que separa la caja de otras cajas.

Demostración visual (lee las etiquetas):

margin (externo)
border (p. ej. 8px solid)
padding (interior)
content (texto, imagen, etc.)

Ejemplo práctico con una caja:

margin: 18px
border: 6px solid
padding: 16px
content: width & height

Clave: ¿Qué mide width y height?

Por defecto (box-sizing: content-box) width/height se aplican solo al content. El padding y border se suman al tamaño total. Con box-sizing: border-box el width incluye content + padding + border (pero no margin).

<div class="caja" style="width:200px;height:100px;padding:20px;border:4px solid">
  /* content = 200x100 */
</div>

/* si box-sizing: content-box */
/* tamaño total = width + 2*padding + 2*border = 200 + 40 + 8 = 248px ancho */

/* si box-sizing: border-box */
/* content ya reducido para encajar en 200x100 incluyendo padding/border */

box-sizing

Valores principales:

  • content-box (por defecto): width y height aplican solo al contenido.
  • border-box: incluye padding y border dentro del width / height. Muy usado porque facilita layouts.
  • padding-box (soporte limitado): incluye padding, pero no border.

Sugerencia práctica (reset común):

:root, * { box-sizing: border-box; }

El valor display y el flujo visual

El comportamiento de una caja en el contexto del documento depende de display. Aquí cubrimos los más importantes y sus efectos:

Valor Descripción Notas prácticas
block Ocupa toda la línea horizontal. Comienza en nueva línea. Anchura por defecto: 100% del contenedor (si no se especifica width).
inline Fluye en línea con texto. Solo acepta margen horizontal y padding horizontal de forma predecible. No respeta width/height en la mayoría de casos; respeta line-height.
inline-block Combinación: se comporta como inline en el flujo, pero acepta width/height. Útil para botones o elementos que deben alinearse en línea y tener tamaño.
none No se representa (oculto), no ocupa espacio. Usado para ocultar elementos sin eliminarlos del DOM.
flex Crea un flex formatting context. Control poderoso de alineación y distribución en un eje. Es el estándar para estructuras unidireccionales (filas/columnas).
grid Layouts bidimensionales (filas + columnas) controlados por la propia caja. Excelente para layouts complejos y áreas macro.
table / table-row / table-cell Valores que imitan el comportamiento de tablas HTML. Útil para alineación tipo tabla sin usar <table>.

Detalles y consideraciones

  • Block formatting context (BFC): ciertos valores (overflow distinto de visible, display:flex/grid, float) crean un BFC que aísla márgenes colapsantes y contiene floats.
  • Márgenes colapsantes: dos márgenes verticales adyacentes (padre-hijo o hermanos) pueden colapsar al mayor de los dos en lugar de sumarse — entender esto evita sorpresas.
  • Replaced elements: <img>, <video>, <input> son "replaced" y su comportamiento con ancho/alto puede diferir.
  • Vertical-align: afecta elementos inline/inline-block; no es lo mismo que flex alignment.
  • Whitespace: los elementos inline/inline-block interpretan espacios en el HTML — a veces hay que eliminarlos con comentarios o CSS.

Resumen práctico / Cheatsheet

Propiedad Qué hace
width, height Definen tamaño del content (a menos que box-sizing sea border-box).
padding Espacio interior; aumenta el tamaño si box-sizing: content-box.
border Contorno. También aumenta tamaño con content-box.
margin Espacio exterior. Puede colapsar verticalmente con otros márgenes.
box-sizing Controla si width incluye padding/border.
display Determina flujo y reglas de formateo (block, inline, inline-block, flex, grid, none...)

Consejo: aplica *, *::before, *::after { box-sizing: border-box; } al comienzo de tu CSS para evitar cálculos mentales y ahorrar bugs de tamaño.
