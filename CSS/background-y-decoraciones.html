Guía completa: Backgrounds y decoraciones (CSS)

Explicaciones, sintaxis y demos para background-color, background-image, background-size, background-position, box-shadow, border-radius y filter.

1. background-color

Define el color de fondo de un elemento. Acepta colores en diferentes formatos:

  • named: red, blue
  • hex: #fff, #1f2937
  • rgb/rgba: rgb(255,0,0), rgba(0,0,0,0.5)
  • hsl/hsla: hsl(200, 80%, 50%)
  • transparent para sin color.
Hex / variable
/* usar variables: */
:root{ --brand: #0ea5a4 }
.box{ background-color:var(--brand); }
#1f2937
.box{ background-color:#1f2937; }
rgba()
.box{ background-color:rgba(14,165,164,0.85); }

Consejo: evita usar colores semitransparentes sobre imágenes si no controlas la luminosidad — usa un linear-gradient encima para garantizar contraste legible.

2. background-image (imágenes y gradientes)

Permite usar imágenes, gradientes o múltiples capas. Sintaxis básica:

background-image: url('imagen.jpg');
background-image: linear-gradient(90deg, #06b6d4, #3b82f6);
background-image: radial-gradient(circle, #fff, #ddd);
background-image: url(a.svg), linear-gradient(...); /* múltiples */
linear-gradient
.hero{ background-image: linear-gradient(135deg, #06b6d4, #3b82f6); }
radial + linear
.badge{ background-image: radial-gradient(circle at 20% 20%, #f97316, transparent 40%), linear-gradient(#fde68a,#fef3c7); }
múltiples capas
/* las capas se pintan de izquierda a derecha. */
.header{ background-image: url('pattern.svg'), linear-gradient(...); }

Orden de capas: la primera capa escrita queda sobre las siguientes (como si apilaras hojas). Para degradados y accesibilidad, coloca el degradado encima de la imagen si quieres oscurecerla o aclararla.

3. background-size y background-position

background-size

cover
background-size: cover; /* llena el contenedor recortando si es necesario */
background-size: contain; /* escala para caber, sin recortar */
background-size: 200px 120px; /* ancho alto */
background-size: auto 50%; /* valores combinados */

background-position

center / contain
background-position: center center; /* keywords: top/right/bottom/left/center */
background-position: 20% 30%; /* porcentajes */
background-position: 10px 20px; /* offsets */
background-position: right top; /* combinaciones */

Para múltiples fondos puedes dar posiciones y tamaños separados por comas en el mismo orden que background-image.

Comportamientos comunes: background-repeat (repeat/no-repeat/round/space), background-attachment (scroll/fixed/local) y background-origin/background-clip (border-box, padding-box, content-box) son propiedades relacionadas que controlan repetición, fijación y recorte.

4. box-shadow

Genera sombras alrededor (o dentro) de elementos. Sintaxis:

box-shadow: [inset?] offset-x offset-y blur-radius spread-radius color;
  • offset-x y offset-y obligatorios.
  • blur-radius (opcional, por defecto 0).
  • spread-radius (opcional) expande/contrae la sombra.
  • inset aplica sombra interior.
  • Puedes especificar múltiples sombras separadas por comas.
Sombra suave
.card{ box-shadow: 0 6px 18px rgba(2,6,23,0.12); }
Múltiples sombras
.modal{ box-shadow: 0 2px 6px rgba(2,6,23,0.08), 0 12px 30px rgba(3,7,18,0.10); }
Inset
.input{ box-shadow: inset 0 2px 8px rgba(2,6,23,0.06); }

Consejo de rendimiento: tener muchas sombras con blur alto en muchos elementos puede afectar la renderización en dispositivos modestos. Si la sombra es decorativa y no dinámica, considera usar una imagen ligera o pseudo-elementos con transformaciones para simular la sombra.

5. border-radius

Controla el redondeo de esquinas. Formas de uso:

border-radius: 8px; /* todas las esquinas iguales */
border-radius: 10px 30px 10px 40px; /* top-left | top-right | bottom-right | bottom-left */
border-radius: 50% ; /* circulo si el elemento es cuadrado */
border-radius: 50% / 30%; /* elíptico */
border-top-left-radius: 12px; /* por esquina */
8px
.chip{ border-radius:8px }
pill
.badge{ border-radius:9999px }
asimétrico
.card{ border-radius: 10px 30px 10px 40px; }
elíptico
.blob{ border-radius:50% / 30% }

Recuerda que overflow afecta si el contenido respeta la esquina redondeada. Usa overflow:hidden si quieres recortar contenido que sale de las esquinas redondeadas.

6. filter

filter aplica efectos visuales a elementos (no solo imágenes). Sintaxis y ejemplos:

filter: blur(4px) brightness(0.8) contrast(1.1) saturate(1.2) grayscale(1) sepia(0.4) hue-rotate(90deg) drop-shadow(4px 6px 8px rgba(0,0,0,0.4));
blur(2px)
.bg{ filter: blur(2px); }
brightness / contrast
.photo{ filter: brightness(0.7) contrast(1.05); }
grayscale(1)
.mono{ filter: grayscale(1); }
.avatar img{ filter: drop-shadow(6px 8px 12px rgba(2,6,23,0.25)); }

Importante: filter crea un nuevo stacking context y puede tener impacto en rendimiento si se aplica a grandes áreas o a muchos elementos. Para animaciones suaves de transform y opacity suelen ser más performantes. Evita animar filtros complejos en móviles si es posible.

7. Combinando propiedades — ejemplos útiles

Shorthand útil:

/* shorthand de background (orden flexible):
background: [background-color] [background-image] [background-position] / [background-size] [background-repeat] [background-origin] [background-clip] [background-attachment];

/* ejemplo */
.header{
  background: linear-gradient(rgba(0,0,0,0.35), rgba(0,0,0,0.35)), url('hero.jpg') center/cover no-repeat;
}

Truco: para superponer un color semitransparente sobre una imagen, coloca un linear-gradient como primera capa y la url() como segunda capa. Ejemplo arriba.

Accesibilidad y rendimiento

  • Comprueba contraste de texto sobre fondos complejos (usa overlays o aumenta contraste).
  • Optimiza imágenes (WEBP, tamaño apropiado, srcset en <img> cuando sea posible).
  • Evita filtros/blur animados en masas; prerenderiza si es posible.
  • Usa variables CSS para esquemas de color y temas.
