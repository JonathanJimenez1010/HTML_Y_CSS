1. ¿Qué es CSS?

CSS (Cascading Style Sheets) es el lenguaje usado para describir la presentación de documentos escritos en HTML (o XML). Mientras HTML define la estructura y el contenido (títulos, párrafos, listas, imágenes), CSS define cómo se ven: colores, fuentes, tamaño, espaciados, disposición en la pantalla, animaciones y efectos visuales.

El término cascading (en cascada) viene de la forma en que el navegador combina reglas desde distintas fuentes (hojas externas, bloques <style>, estilos inline) y resuelve conflictos según un conjunto de reglas (especificidad, orden y !important).

Tip: piensa en HTML como el esqueleto y el contenido; CSS como la ropa y la estética que lo hace usable y atractivo.

2. Cómo se enlaza al HTML

Hay tres formas de aplicar CSS a un documento HTML. Cada una tiene usos concretos:

a) Inline

Se aplica directamente en el atributo style del elemento HTML. Es útil para pruebas rápidas o para sobrescribir un estilo muy concreto, pero dificulta el mantenimiento si se usa masivamente.

<!-- Inline: estilo directamente en la etiqueta -->
<h1 style="color: #0b61ff; font-size: 28px;">Hola mundo</h1>

Evita usar inline en proyectos grandes. Rompe la separación de responsabilidades.

b) Internal (estilos en la misma página)

Incluyes un bloque <style> en el <head>. Se usa para páginas únicas o demostraciones.

<head>
  <style>
    body { font-family: Arial, sans-serif; }
    .destacado { background: #fffae6; padding: 8px; }
  </style>
</head>

c) External (archivo .css separado)

La práctica recomendada: separar CSS en archivos .css y enlazarlos con <link rel="stylesheet" href="styles.css">. Esto facilita mantenimiento, caché y reutilización entre páginas.

<!-- En el <head> -->
<link rel="stylesheet" href="styles.css">

/* styles.css */
body { font-family: Arial, sans-serif; }

3. Selectores básicos

Los selectores determinan qué elementos recibirán las reglas. Los tres más importantes al empezar son:

  • Selector de elemento (tag): aplica a todas las etiquetas del mismo tipo p, h1, ul, etc.
  • Selector de clase (.clase): reutilizable, aplica a cualquier elemento con esa clase.
  • Selector de id (#id): único por página (debe usarse en un solo elemento).
Elemento 
p {
  color: #334155;
  font-size: 16px;
}
Clase 
.boton {
  background: #0ea5a5;
  color: white;
  padding: 8px 12px;
  border-radius: 6px;
}
ID 
#hero-title {
  font-size: 32px;
  font-weight: 700;
}

Buenas prácticas con selectores

  • Usa .clase para estilos reutilizables y componentes.
  • Evita estilos con selectores demasiado específicos (por ejemplo, div > ul li a.btn) salvo que sea necesario.
  • Usa IDs para comportamientos únicos (JS targeting), no como la primera opción para estilos.
  • Prefiere nombres semánticos: .card, .nav-item, .btn-primary.

4. Propiedad — valor y sintaxis

La estructura de una regla CSS:

selector {
  propiedad: valor;
  otra-propiedad: otro-valor;
}

Ejemplo:

.card {
  background: #fff;
  padding: 12px;
  border-radius: 10px;
}

Algunas propiedades comunes

  • color — color del texto
  • background / background-color — color o imagen de fondo
  • font-size, font-family — tipografía
  • padding, margin — espaciamiento interno / externo
  • display, position — control del layout

5. Especificidad y cascada (resumen práctico)

Cuando varias reglas afectan a un mismo elemento, el navegador decide cuál aplicar usando especificidad y orden. Regla simple:

  1. Estilos inline (atributo style="...") tienen mayor peso.
  2. IDs (#id) son más específicos que clases (.clase).
  3. Clases, pseudoclases y atributos (.clase, :hover, [type="text"]) comparten un nivel.
  4. Selectores de elemento (p, div) tienen menor especificidad.
  5. Si la especificidad es igual, gana la regla que aparece después en el CSS (última declarada).
Ejemplo: Si tienes p { color: blue; } y .lead { color: red; } y el párrafo tiene la clase lead, el color será rojo porque la clase es más específica que el selector de elemento.

6. Herencia

Algunas propiedades CSS se heredan por defecto a los hijos (por ejemplo color, font-family), mientras que otras no (por ejemplo margin, padding). Esto puede ser útil: establece estilos base en body y muchos elementos los heredarán.

body { font-family: 'Inter', Arial, sans-serif; color: #111; }
/* Los elementos de texto normalmente heredan font-family y color */

7. Box model explicado

El box model describe cómo se calcula el tamaño de los elementos en CSS. Está compuesto por:

  • Content — el contenido (texto, imagen)
  • Padding — espacio interno alrededor del contenido
  • Border — borde alrededor del padding
  • Margin — espacio externo que separa con otros elementos
/* Ejemplo: caja con padding, borde y margen */
.box {
  width: 200px;           /* ancho del contenido */
  padding: 16px;          /* espacio interno */
  border: 2px solid #ccc; /* borde */
  margin: 12px;           /* separación externa */
}

Importante: la propiedad box-sizing afecta si el width incluye o no el padding y el border. Recomendación moderna:

*, *::before, *::after { box-sizing: border-box; }

Con border-box, el width incluye padding y border — facilita el diseño responsivo.

8. Consejos de estilo y buenas prácticas

  • Organiza tu CSS: variables en :root, componentes (clases) por secciones, utilidades al final.
  • Usa variables CSS para colores y tamaños: --primary: #0ea5a5; y color: var(--primary);.
  • Prefiere nombres semánticos y consistentes (p. ej. BEM: .card__title, .card--large).
  • Minimiza el uso de !important — suele indicar problemas de arquitectura.
  • Usa external CSS en producción para aprovechar caché y limpieza del HTML.

9. Errores comunes y cómo resolverlos

  • Olvidar ; (punto y coma): provoca que la siguiente propiedad sea ignorada. Revisa la consola del inspector.
  • Selector mal escrito: revisa espacios y caracteres especiales (.btn-primary vs .btn_primary).
  • Especificidad inesperada: si un estilo no se aplica, intenta inspeccionar el elemento para ver qué regla lo sobrescribe.
  • Rutas de archivo incorrectas al usar <link href="styles.css"> — la ruta es relativa al HTML que carga el CSS.

10. Ejercicios recomendados y retos

  1. Crear una tarjeta (.card) con imagen, título y descripción. Hacerla responsive.
  2. Diseñar un botón con estados: normal, hover, active y disabled. Usar variables CSS para colores.
  3. Recrear un header simple con logo a la izquierda y navegación a la derecha usando Flexbox (no lo cubrí aquí en profundidad — es lo siguiente a estudiar).
  4. Usar el inspector del navegador para modificar estilos y entender la cascada en tiempo real.

Apéndice: Ejemplo final (lista para copiar)

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Ejemplo CSS Básico</title>
  <style>
    :root{
      --bg: #f1f5f9;
      --card-bg: #ffffff;
      --primary: #0b61ff;
      --accent: #0ea5a5;
      --muted: #334155;
    }

    * , *::before, *::after { box-sizing: border-box; }

    body{
      font-family: Arial, Helvetica, sans-serif;
      background: var(--bg);
      margin: 0; padding: 18px; color: var(--muted);
    }

    .card{
      background: var(--card-bg);
      padding: 12px;
      border-radius: 10px;
      box-shadow: 0 2px 8px rgba(11,17,34,0.04);
      max-width: 420px;
    }

    .btn{
      display:inline-block;padding:8px 12px;border-radius:6px;background:var(--accent);color:white;text-decoration:none;
    }

  </style>
</head>
<body>

  <h1>Tarjeta de ejemplo</h1>
  <div class="card">
    <h2>Título de la tarjeta</h2>
    <p>Descripción breve para ilustrar cómo usar clases, propiedades y el box model.</p>
    <a href="#" class="btn">Acción</a>
  </div>

</body>
</html>
