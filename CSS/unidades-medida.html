Guía completa: Unidades de medida en CSS

Explicación, ejemplos, buenas prácticas y trucos — ordenado exactamente como pediste

Introducción

CSS ofrece varias unidades para medir tamaños (tipografías, boxes, márgenes, media queries, imágenes, etc.). Elegir la unidad correcta mejora la accesibilidad, el rendimiento y la adaptabilidad del diseño.

A continuación verás cada grupo en este orden: Absolutas: px, Relativas al contenedor: %, Relativas a tipografía: em, rem, Relativas a la ventana: vw, vh, vmin, vmax, y Funciones: clamp(), min(), max().

1) Absolutas: px

Qué es: px (píxeles) mide en píxeles de referencia del dispositivo (unidad más usada para controles precisos).

Características

  • Valor absoluto en el contexto de la pantalla — resultado consistente pixel a pixel.
  • Ideal para bordes, iconos y layouts donde se requiere precisión.
  • No escala automáticamente según la tipografía raíz del usuario (salvo zoom del navegador o ajustes de OS).

Ejemplos

/* tamaño fijo */
.box { width: 300px; height: 200px; }
.text { font-size: 16px; }
Usa px cuando necesites control pixel-perfect; evita usar sólo px para tipografías en diseños responsivos y cuando quieras respetar la preferencia de tamaño del usuario.

Problemas/Pitfalls

  • Puede romper la accesibilidad: usuarios con configuraciones de texto más grande pueden no ver cambios si todo está en px.
  • En pantallas de alta densidad (retina) hay más píxeles físicos, pero CSS sigue manejando px de forma consistente.

2) Relativas al contenedor: %

Qué es: Porcentaje relativo a un contenedor (normalmente el elemento padre) o a la referencia de la propiedad especificada.

Características

  • Muy útil para layouts fluidos: ancho, padding vertical/horizontal (en ciertos casos), imágenes y posicionamiento.
  • El valor depende del contexto: width:50% en un hijo = 50% del ancho del padre.

Ejemplos

.card { width: 80%; max-width: 1200px; margin: 0 auto; }
.child { width: 50%; }
.img { width: 100%; height: auto; /* imagen responsive */ }
En reglas de altura, los porcentajes se calculan respecto al height del padre si éste tiene una altura establecida; si no, puede dar resultados inesperados.

Problemas/Pitfalls

  • Si el padre no tiene una dimensión explícita para la propiedad, el porcentaje puede no funcionar como esperas (ej. porcentaje de altura sin altura padre definida).
  • El flujo y la herencia pueden afectar el cálculo; es contextual.

3) Relativas a la tipografía: em y rem

Concepto clave: ambas dependen del tamaño de fuente, pero de diferente referencia.

em

Referencia: el tamaño de fuente del elemento en cuestión (o su elemento padre en propiedades que heredan).

Comportamiento: es compuesta — se multiplica por la cadena de tamaños de fuente hasta la raíz si hay cambios intermedios.

Ejemplo y explosión

html { font-size: 16px; }
.container { font-size: 1.25em; /* 1.25 * 16 = 20px */ }
.container p { font-size: 0.8em; /* 0.8 * 20 = 16px */ }

Nota: atención con el efecto acumulativo (anidado). Esto lo hace poderoso, pero a veces impredecible.

rem

Referencia: el tamaño de fuente de la raíz (html), es decir, root em.

Ejemplo

html { font-size: 16px; }
h1 { font-size: 2rem; /* 32px */ }
.small { font-size: 0.875rem; /* 14px */ }

Regla práctica: usa rem para tamaños globales (tipografías base, paddings globales) y em para elementos que deban escalar respecto a su propio contexto (por ejemplo control de componente).

Accesibilidad

  • Si un usuario cambia el tamaño base del navegador (por ejemplo 18px), rem respeta esa preferencia; es bueno para accesibilidad.
  • em también respeta, pero puede comportarse diferente si hay anidamientos y cambios intermedios.

Problemas/Pitfalls

  • Confusión por anidamiento con em. Mantén reglas simples o documenta las dependencias.
  • Si cambias html{font-size} con porcentajes, afectará todo el sistema si usas rem.

4) Relativas a la ventana (viewport): vw, vh, vmin, vmax

Estas unidades se basan en el tamaño de la ventana gráfica (viewport). Son excelentes para diseños fluidos y efectos que relacionan tamaño con la ventana del usuario.

vw y vh

  • 1vw = 1% del ancho del viewport.
  • 1vh = 1% de la altura del viewport.
.hero { height: 100vh; /* ocupa toda la altura visible */ }
.big-title { font-size: 8vw; /* tamaño proporcional al ancho de la ventana */ }

vmin y vmax

  • vmin = 1% del menor lado del viewport (ancho o alto).
  • vmax = 1% del mayor lado del viewport.
/* usa vmin si quieres que algo escale según el lado más pequeño (útil para mantener proporción) */
.box { width: 50vmin; height: 50vmin; }

Problemas/Pitfalls

  • En móviles, la UI del navegador (barra de direcciones) puede aparecer/ocultar y cambiar el valor de vh; esto puede causar saltos visuales si dependes de 100vh. Existen técnicas y variables CSS modernas para mitigar esto (ej. usar height: 100svh cuando está disponible o calcular con JS).
  • Remarca que vw/vh no tienen en cuenta la escala de la tipografía establecida por el usuario (pero sí cambian en zoom de página).

5) Funciones: clamp(), min(), max()

Estas funciones permiten expresar valores adaptativos combinando unidades. Son poderosas para crear tipografías y layouts fluidos sin necesidad de media queries.

clamp(min, preferred, max)

Devuelve un valor entre min y max que intenta usar preferred si está dentro del rango. Muy útil para tipografías fluidas.

/* Ejemplo: título que escala con el ancho, pero tiene límites */
h1 { font-size: clamp(1.2rem, 2vw + 1rem, 3rem); }
/* mínimo 1.2rem, preferible (2vw + 1rem), máximo 3rem */

min() y max()

Seleccionan el menor o mayor valor entre los argumentos.

.box { width: min(80%, 900px); /* ancho será el menor entre 80% del contenedor o 900px */ }
.container { padding: max(1rem, 2vw); /* padding será el mayor entre 1rem o 2vw */ }

Casos de uso y ventajas

  • clamp() es ideal para tipografías fluidas, paddings y gaps que deben crecer hasta un tope.
  • min()/max() combinadas con vw/rem permiten reglas simples que reaccionan al viewport sin media queries.

Compatibilidad

Estas funciones tienen buena compatibilidad en navegadores modernos. Revisa caniuse si necesitas soporte para navegadores muy antiguos.

Comparaciones rápidas

Unidad Referencia Mejor uso Limitaciones
px pixeles de dispositivo Bordes, iconos, precisión No escala con la preferencia tipográfica del usuario
% contenedor (padre) anchos fluidos, imágenes responsive Depende del padre; cuidado con alturas
em tamaño del elemento componentes que escalen por contexto Anidamiento puede causar acumulación
rem tamaño raíz (html) sistemas de tipografía y espaciado global afecta todo si cambias la raíz
vw/vh/vmin/vmax viewport layouts dependientes del viewport, hero full-screen Móviles: UI del navegador afecta vh
clamp/min/max funciones combinadas tipografía y espacios fluidos sin media queries requiere navegadores modernos

Buenas prácticas y recomendaciones

  1. Adopta rem para la base tipográfica: define html{font-size} y usa rem para tamaños globales (headers, body base, paddings globales).
  2. Usa em para componentes (por ejemplo controles que deben escalar según su contexto).
  3. Evita mezclar demasiado px para tipografías si te importa la accesibilidad; reserva px para elementos UI fijos.
  4. Combina clamp() + vw/rem para crear tipografías fluidas y controladas.
  5. Para ancho de layout, prefiere porcentajes con max-width y límites en px o rem.
  6. Ten en cuenta móviles: evita usar exactamente 100vh para secciones que deben ocupar la pantalla completa en móviles — considera alternativas (min-height, svh cuando esté disponible).

Ejemplos prácticos (plantillas)

Copia y prueba estos fragmentos.

Tipografía fluida con clamp() 
html { font-size: 16px; }
h1 { font-size: clamp(1.5rem, 4vw, 3rem); }
body { font-size: 1rem; line-height: 1.6; }
Layout con % y max-width 
.container { width: 90%; max-width: 1200px; margin: 0 auto; }
.sidebar { width: 30%; }
.content { width: 70%; }
Componente que usa em para escalar localmente 
.card { font-size: 1rem; /* base local */ padding: 1.25em; }
.card h3 { font-size: 1.25em; /* relativo a la card */ }
Hero full-screen con vh (cuidado en móviles) 
.hero { height: 100vh; display:flex; align-items:center; justify-content:center; }
/* alternativa segura: min-height: 100vh; fallback con JS para móviles. */

Trucos y tips avanzados

  • Si usas clamp() con vw, puedes evitar media queries para tamaños intermedios.
  • Combina min() con 100% y px para evitar que un elemento crezca más allá de cierto tope: width: min(100%, 1200px).
  • Para el problema de 100vh en móviles, hoy existen unidades dinámicas como svh, lvh, dvh — úsalas si el navegador lo soporta, y deja fallback con vh.
  • Cuando calcules transformaciones y subpíxeles, ten en cuenta el antialiasing y el redondeo del motor de render del navegador; a veces es preferible usar valores pares o escalas con transform para evitar bordes borrosos.
