Transiciones, Transformaciones y Animaciones en CSS

Esta página recoge una referencia completa, ejemplos prácticos y buenas prácticas sobre transition, transform (scale, rotate, translate...) y animation / @keyframes. Incluye demos interactivos y recomendaciones de accesibilidad y rendimiento.

1) transition — transiciones suaves

Las transiciones permiten interpolar cambios entre un estado inicial y otro de propiedades CSS cuando estas cambian (por hover, foco, cambio de clase, JS...).

Sintaxis

/* forma larga */
transition-property: property;
transition-duration: time;
transition-timing-function: timing-function;
transition-delay: time;

/* shorthand */
transition: property duration timing-function delay;

Ejemplos útiles

/* transición simple al cambiar transform y box-shadow */
.trans-square{ transition: transform 300ms ease, box-shadow 200ms ease; }

/* transición para cualquier propiedad modificada: */
.btn{ transition: all 180ms cubic-bezier(.2,.8,.2,1); }

Consejos

  • Prefiere declarar las propiedades que animarás en vez de usar all (mejor rendimiento).
  • Usa will-change con moderación para avisar al navegador de próximas animaciones y lograr aceleración por GPU.
  • Combina con prefers-reduced-motion para respetar la accesibilidad.

2) transform — scale, rotate, translate y combinaciones

transform permite aplicar transformaciones 2D/3D al elemento. Las transformaciones no afectan el flujo del documento (el elemento se redibuja transformado).

Funciones principales

scale(sx [, sy])       /* escala en X e Y */
rotate()          /* rotación en grados (deg) o rad */
translate(tx [, ty])    /* movimiento en px, %, em, etc. */
translateX(), translateY(), translateZ()

/* 3D */
rotateX(), rotateY(), rotateZ(), perspective(), translateZ()

/* combinarlas: */
transform: translateX(10px) rotate(12deg) scale(1.08);

Ejemplos prácticos

/* Scale (agrandar/reducir) */
.card:hover{ transform: scale(1.05); }

/* Rotate (girar) */
.icon{ transform: rotate(45deg); }

/* Translate (mover) */
.toast{ transform: translateY(-8px); }

/* Combinado */
.btn:hover{ transform: translateY(-6px) rotate(-2deg) scale(1.03); }

transform-origin

Controla el punto de referencia para rotaciones y escalados (por defecto: 50% 50% — centro). Ejemplos:

transform-origin: center;
transform-origin: 10% 90%; /* x y */

Notas sobre 2D vs 3D y rendimiento

  • Transformaciones 3D (ej. translateZ o rotateY) pueden activar la GPU para un render más suave.
  • Evita animar propiedades que disparen reflow (layout) como width o left si buscas suavidad; usa transform y opacity cuando sea posible.

3) animation y @keyframes

Las animaciones te permiten definir secuencias complejas cuadro-a-cuadro con @keyframes y aplicarlas mediante propiedades de animación.

Sintaxis y propiedades principales

@keyframes nombre { 0% { ... } 50% { ... } 100% { ... } }

/* propiedades */
animation-name: nombre;
animation-duration: 1s; /* obligatorio para que ocurra */
animation-timing-function: ease|linear|steps()|cubic-bezier(...);
animation-delay: 0s;
animation-iteration-count: 1|infinite|n;
animation-direction: normal|reverse|alternate|alternate-reverse;
animation-fill-mode: none|forwards|backwards|both;
animation-play-state: running|paused;

/* shorthand */
animation:       ;

Ejemplos de @keyframes

/* 1 - Giro continuo */
@keyframes spin { to { transform: rotate(360deg); } }

/* 2 - Rebote simple */
@keyframes bounce { 0%,100% { transform: translateY(0) } 50% { transform: translateY(-18px) } }

/* 3 - Slide in */
@keyframes slideIn { from { opacity:0; transform: translateY(10px) } to { opacity:1; transform:none } }

/* 4 - "Typewriter" con steps() */
@keyframes typing { from { width:0 } to { width: 16ch } }

Propiedades importantes explicadas

  • animation-duration: tiempo total de la animación.
  • animation-timing-function: controla la aceleración (p. ej. cubic-bezier(.2,.9,.25,1), steps(5, end)).
  • animation-iteration-count: cuántas veces se ejecuta (o infinite).
  • animation-fill-mode: qué estilo mantiene el elemento antes o después de la animación (forwards hace que el elemento conserve el estado final).
  • animation-play-state: permite pausar/reanudar con JS.

Uso de steps() para animaciones tipo frame-by-frame

/* útil para sprites o efecto máquina de escribir */
.element{ animation: blink 1s steps(2, end) infinite }

Combinar múltiples animaciones

.el{ animation: spin 1s linear infinite, pulse 2s ease-in-out infinite; }

Control desde JavaScript

const el = document.querySelector('.box');
el.style.animationPlayState = 'paused';
el.style.animationPlayState = 'running';

Accesibilidad

Respeta la preferencia del usuario con @media (prefers-reduced-motion: reduce) para desactivar o simplificar animaciones y evitar mareos o molestias.

4) Mejores prácticas y rendimiento

  • Prioriza animar transform y opacity por rendimiento.
  • Usa will-change solo en elementos que realmente lo necesiten y no de forma global: will-change: transform;.
  • Evita usar demasiadas animaciones simultáneas en dispositivos con poca potencia.
  • Proporciona controles para pausar animaciones intensas si el caso de uso lo requiere.
  • Usa animation-fill-mode: both para animaciones de entrada si quieres mantener el estado final interactivamente.

Compatibilidad

Las propiedades principales están ampliamente soportadas. En casos muy antiguos podrías necesitar prefijos (-webkit-) para ciertos navegadores viejos, pero hoy en día en la mayoría de proyectos modernos no es necesario.

5) Snippets listos para copiar

/* Botón con transición y transform */
.btn{ transition: transform 220ms cubic-bezier(.2,.9,.25,1), box-shadow 220ms; }
.btn:hover{ transform: translateY(-6px) scale(1.03); box-shadow: 0 12px 30px rgba(2,6,23,0.6); }

/* Reproducción controlada desde JS */
.el{ animation: fadeIn 600ms ease both; }
.el.paused{ animation-play-state: paused; }

/* Prefers reduced motion */
@media (prefers-reduced-motion: reduce){ .btn, .el{ transition:none; animation:none } }
