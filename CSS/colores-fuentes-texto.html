1. Tipos de colores (hex, rgb, hsl)

Principales formatos y cuándo usarlos:

  • Hexadecimal — ejemplo: #ff6b6b. Compacto y muy usado en diseño.
  • RGB / RGBA — ejemplo: rgb(255,107,107) o con alfa rgba(255,107,107,0.6). Útil cuando quieres controlar opacidad sin separar propiedad.
  • HSL / HSLA — ejemplo: hsl(0,100%,64%) o hsla(0,100%,64%,0.6). Excelente para ajustar tonalidad, saturación o luminosidad de forma intuitiva.
  • Valores CSS modernos — color-mix(), lab() (soporte variable), y system color keywords.

Consejos:

  1. Usa HSL para generar paletas y variantes (p. ej. cambiar sólo la luminosidad para obtener tonos más oscuros/claro).
  2. Usa variables CSS para mantener consistencia: :root{--primary:#0b76ff}.
  3. Para accesibilidad, comprueba contraste (ratio) entre texto y fondo.

Fragmentos

<!-- Hex -->
.element{ background:#ff6b6b; }

<!-- RGB/RGBA -->
.element{ color:rgb(20,20,20); background:rgba(11,118,255,0.12); }

<!-- HSL/HSLA -->
.element{ background:hsl(210,100%,56%); }

<!-- Variable CSS -->
:root{ --primary: hsl(210,100%,56%); }
.btn{ background:var(--primary); }

2. font-family, font-size, font-weight

font-family

Regla: siempre define una pila (stack). Ejemplo:

body{
  font-family: 'Inter', system-ui, -apple-system, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif;
}

Opciones:

  • Genéricos: serif, sans-serif, monospace, cursive, fantasy, system-ui
  • Coloca fuentes locales o de web (Google Fonts) al inicio de la lista.

font-size

Unidades comunes y recomendaciones:

  • px — fijo, predecible. Bueno para elementos que no deben escalar.
  • rem — relativo al font-size raíz (html). Ideal para escalabilidad y accesibilidad.
  • em — relativo al tamaño del elemento padre (útil en componentes).
  • clamp() — para tamaños fluidos: font-size: clamp(1rem, 2.5vw, 1.25rem);.

font-weight

Valores:

  • Palabras clave: normal (400), bold (700), bolder, lighter.
  • Números: 100 a 900 (si la familia las soporta).
h1{ font-family: 'Montserrat', sans-serif; font-size:clamp(1.6rem,4vw,2.2rem); font-weight:700; }
small{ font-size:0.875rem; font-weight:400; }

Ejemplo: Título — Poppins 700

Párrafo demostrativo — Inter 400. Usa rem para accesibilidad: 1rem = tamaño raíz.

Recuerda incluir la fuente en el <head> (Google Fonts) o guardarla localmente.

3. Google Fonts — cómo incluir y buenas prácticas

Pasos rápidos:

  1. En Google Fonts selecciona la familia y los pesos que usarás (no pidas todos los pesos: sólo los necesarios para rendimiento).
  2. Incluye el <link> en el <head> o usa @import en CSS (preferible el <link> por rendimiento).
  3. Usa font-display: swap; para mejorar experiencia al cargar fuentes. 
<!-- Head -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap" rel="stylesheet">

<!-- CSS -->
body{ font-family:'Inter', system-ui, -apple-system, 'Segoe UI', Roboto, sans-serif; }
/* si usas @import: @import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap'); */

Consejos: elegir pocas familias (1–2) y sólo los pesos necesarios (p. ej. 400,600,700). Para íconos usa icon fonts o SVGs en lugar de familias grandes.

Fragmento que demuestra font-display: swap (aplica desde Google Fonts cuando agregas &display=swap al enlace):

/* Google ya aplica display=swap si lo pides en la URL */
@font-face{
  font-family:'MiFuente';
  src: url(...);
  font-display: swap;
}

4. Espaciado: line-height, letter-spacing

line-height

Controla altura de línea (espacio vertical entre líneas de texto):

  • Unitless (recomendado): line-height: 1.5; — escala con el tamaño de fuente y evita acumulación inesperada.
  • Unidades: line-height: 24px; — útil cuando quieres un valor exacto.
  • Valores típicos: 1 (compacto), 1.25–1.6 (texto de lectura), 2+ (captions o diseño aireado).

letter-spacing

Espaciado entre caracteres:

  • Valores en em o px: letter-spacing: 0.02em;
  • Se usa en títulos, mayúsculas, o para ajustar kerning en tipografías web.
  • Evitar espacios positivos grandes en texto de párrafo porque reduce legibilidad.
p{ line-height:1.6; letter-spacing:0.01em; }
h1{ letter-spacing:0.02em; line-height:1.15; }

5. text-align (alineación de texto)

Valores comunes:

  • left — alineación a la izquierda (por defecto en lenguajes LTR)
  • right — alineación a la derecha
  • center — centrado horizontal
  • justify — justificado (ajusta espacios entre palabras para alinear ambos lados) 
.card p{ text-align:left; }\.header{ text-align:center; }
.article{ text-align:justify; }

Título centrado

Párrafo justificado de muestra. El texto justificado alinea ambos márgenes y crea un bloque limpio, pero puede generar espacios irregulares entre palabras en columnas estrechas.
